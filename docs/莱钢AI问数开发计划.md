## 1. 总体计划概览

### 1.1 目标

**第一期目标：2026年5月25日正式上线**，实现以下核心能力：

- 四大数据域（财务、销售、生产、采购）Excel 文件上传 → 自动解析 → 清洗入库
- 自然语言中文提问 → Text-to-SQL → 结果展示（表格 / 图表）
- 多轮对话、数据时效标注、用户反馈
- RBAC 权限管理 + 操作审计日志
- 管理后台：数据源管理、SQL 日志、准确率统计

### 1.2 不在第一期范围

- ERP / MES 等现有系统直连
- 移动端 App
- AD 域集成（统一身份认证）
- 实时数据流接入

### 1.3 关键数字

| 指标 | 目标 |
|------|------|
| SQL 准确率（上线前） | ≥ 85%（简单 ≥ 95%，中等 ≥ 82%，复杂 ≥ 65%）|
| SQL 执行成功率 | ≥ 95% |
| 查询 P95 响应时间 | 简单查询 ≤ 8s，复杂查询 ≤ 15s |
| 系统并发支撑 | ≥ 30 用户同时在线 |
| 单元测试覆盖率 | 安全模块 100%，核心模块 ≥ 85%，其余 ≥ 75% |
| UAT 通过率 | 关键场景 ≥ 90% |

## 4. 模块开发任务拆解

### 4.1 模块依赖关系与开发顺序

```
安全模块（sql_validator / rbac / row_filter / audit）
    ↓ 优先完成（前置依赖）
LLM 调用层（base / qianwen / wenxin / router）
    ↓ 并行
知识库（embedding / dictionary / cache）
    ↓ 并行
Excel 解析引擎（excel_parser）
    ↓ 依赖 LLM 调用层
字段映射（field_mapper）         数据清洗（data_cleaner）
    ↓ 依赖安全模块 + LLM + 知识库
Text-to-SQL 引擎（text_to_sql / prompt_builder）
    ↓ 依赖安全模块
查询执行器（sql_executor）       结果格式化（result_formatter / nlg）
    ↓ 全部核心模块完成后
对话管理（conversation）
    ↓ 全部核心模块完成后
REST API 层（auth / datasource / chat / admin）
    ↓ 后端完成后
前端（Login / Chat / Datasource / Admin）
```

### 4.2 关键路径识别

**关键路径（延误直接影响 M1）：**

`Excel 解析引擎` → `字段映射` → `数据清洗入库` → `Text-to-SQL 引擎` → `查询执行` → `API 联调`

- Excel 解析是 Text-to-SQL 的数据前提，必须 4/19 前完成
- Text-to-SQL 引擎是核心，必须 4/26 前基本可用
- API 联调依赖后端全部模块，必须 5/4 前完成

**可并行的工作流：**
- 安全模块 + LLM 调用层（04/10 ～ 04/17 并行开发）
- Excel 解析 + 知识库构建（04/10 ～ 04/24 并行开发）
- 前端开发（04/20 启动后可与后端并行，通过 Mock 接口）

## 7. 风险与应对

| 风险编号 | 风险描述 | 等级 | 触发信号 | 应对措施 |
|---------|---------|------|---------|---------|
| **R01** | Excel 格式不规范导致解析失败率高 | 中高 | M1 前解析成功率 < 80% | 提前下发规范模板；字段映射提供人工兜底界面；解析器增加智能错误提示 |
| **R02** | 业务部门数据提交延迟（影响数据准备阶段）| 中高 | 04/30 仍有数据域未提交 | PM 提前两周启动协调；设置数据交付 deadline 04/27；先用样本数据推进测试 |
| **R03** | 跨 Excel 关联查询准确率偏低 | 中高 | M2 复杂问题准确率 < 55% | 第一期优先保障单表查询质量；设计关联键配置机制；积累跨表 Few-shot 示例 |
| **R04** | SQL 准确率在 M2 不达标 | 中 | M2 总体准确率 < 80% | Prompt 专项调优；补充高质量 Few-shot；数据字典质量提升；若差距大则延迟上线 |
| **R05** | LLM API 调用成本超支 | 低中 | 日均 Token 超过预算 80% | 开启语义缓存（相似度 0.92）；设置 Token 日预算告警；结果缓存 5 分钟 |
| **R06** | 生产环境服务器端口冲突（与其他项目）| 低 | 启动脚本端口检查失败 | 严格执行 CLAUDE.md 第3章端口分配；start.sh 内置检查；提前与其他项目确认 |
| **R07** | 数据安全事故（真实数据发给 LLM API）| 低（高影响）| 审计发现 payload 含真实数据 | 单元测试覆盖 LLM payload 脱敏校验；脱敏模块独立且有专项测试；CI 自动检查 |
| **R08** | 前端开发工期滞后（依赖后端接口稳定）| 中 | 04/30 接口仍有较多变更 | 优先稳定核心接口（chat / datasource）；前端用 Mock 数据并行开发；05/01 前接口冻结 |

**应急响应原则：**
- P0 风险（数据安全）：立即停机、通知负责人、溯源审计
- P1 风险（核心功能不可用）：24 小时内响应修复
- 准确率不达标：不延迟关联里程碑，但延迟上线日期，优先质量
